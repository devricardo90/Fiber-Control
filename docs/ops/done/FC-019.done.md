## Objetivo e escopo
- alinhar o ambiente local de `apps/web` e `apps/api` a `web.fiber-control.localhost` e `api.fiber-control.localhost`
- alinhar `APP_URL`, `NEXT_PUBLIC_APP_URL`, `NEXT_PUBLIC_API_URL` e `CORS_ORIGIN`
- ajustar o minimo necessario para subir localmente com host nomeado
- manter fallback documentado para `localhost:porta`

## Fora de escopo
- Portless generico para outros projetos
- novos subdominios alem de `web` e `api`
- refactor amplo de configuracao
- mudancas funcionais de produto
- infraestrutura nova fora do minimo necessario para o ambiente atual

## O que foi consolidado
- `apps/web/.env.local` e `apps/web/.env.example` passaram a usar `web.fiber-control.localhost` e `api.fiber-control.localhost`
- `apps/api/.env` e `apps/api/.env.example` passaram a expor `APP_URL` e `CORS_ORIGIN` coerentes com o host nomeado
- o script `dev` do web passou a subir com `--hostname web.fiber-control.localhost --port 3000`
- a baseline de ambiente no frontend deixou de hardcodear `localhost:3000`
- o fallback para `localhost:porta` permaneceu registrado na governanca
